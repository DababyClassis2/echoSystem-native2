# Contributing Guide

## Branch Strategy
- `main` → stable, production-ready
- `dev` → active development
- Feature branches: `feature/<name>`

## Pull Request Rules
- Must pass CI
- Must include documentation updates
- Must include tests for new logic

## Coding Standards
- Kotlin style guide
- Compose best practices
- Hilt for DI
- MVVM for UI state

## Commit Message Format
- `fix: ...`
- `feat: ...`
- `refactor: ...`
- `docs: ...`
